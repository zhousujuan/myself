P00 市场选择

00 市场选择

一切的起点。选错国家 = 后面 9 步白做。预计单次决策 4–8 小时（含数据收集）。

① 输入 / 前置

  • 已决定要做的产品类目（不是从零选品）。
  • 初步预算 / 团队语言能力 / 现有客户分布。
  • 对各市场有最基本印象（不是完全陌生）。

② 关键动作

  1. 列 5–8 个候选国家。
  2. 用 ITC Trade Map 看市场规模和增长趋势。
  3. 评估关税 / 认证 / 物流 / 货币 / 政治风险。
  4. 看竞争密度（Google 搜 "[产品] [国家]" 看前 20 条）。
  5. 收敛到 1–3 个市场，写下"为什么选"3 句话。

③ 工具 + 数据源（点击复制）

ITC Trade Map (trademap.org) — 双边进出口数据 + 趋势 WTO Tariff Database — 关税查询 Statista / IBISWorld — 行业报告 + 市场规模 Google Trends — 关键词热度地理分布 World Bank Doing Business — 营商环境 + 跨境物流 调 A07 海关数据反查 → 验证候选国是否真在进口

④ 输出 / 交付物

  • 1–3 个目标国家
  • 每国的价格带 + 主渠道类型
  • 关税 + 认证门槛清单
  • 物流 + 货币 + 政治风险评估
  • "为什么选这些"3 句话备忘录

⑤ 完成标准

⑥ 常见卡点

  • 选 5+ 国分散精力 → 哪个都做不深。
  • 没考虑认证门槛 → 报价后才发现进不了。
  • 没看汇率风险 → 利润被汇率吃掉。
  • 凭"听说市场好"选 → 数据不支撑。
  • 跳过这一步直接找客户 → ICP 漂浮，转化低。

P01 ICP 定义

01 ICP 定义

决定后面 9 步招呼谁。30 秒说不清就是没定义清。预计 2–4 小时收敛画像。

① 输入 / 前置

  • 00 已选定目标市场。
  • 对自家产品差异化 / 价位 / MOQ 清楚。
  • 历史客户画像（若有）作为基础假设。

② 关键动作

  1. 假设 2–3 个客户类型（进口商 / 品牌 / 连锁 / 工程 / 代理）。
  2. 用 A07 海关 + A01 Google + A02 LinkedIn 验证存在性。
  3. 收敛到 1–2 个最优画像。
  4. 画决策人地图（采购 / 老板 / 类目 / 技术 / 财务谁拍板）。
  5. 写一句 30 秒能说清的 ICP 公式。

③ 工具 + ICP 公式模板（点击复制）

ICP 公式：[国家] 做 [业务/品类] 的 [客户类型]，规模 [N 人/N 万 USD]，采购模式 [自购/招标/项目]，决策人 [角色] 示例：德国做厨房家电零售的 mid-size 进口商，5-50 人，月采购 5-20 万 EUR，决策人是 Buyer / Procurement Manager 验证方法：用 ICP 公式去 LinkedIn Sales Nav 搜 → 能搜出 ≥30 家说明存在 决策人地图：列出 3 类（拍板人 / 影响人 / 阻碍人）+ 他们关心什么 痛点假设：列 3 个对方"为什么会换供应商"的具体理由

ICP 太宽（"任何采购"）= 0% 转化。太窄（只 5 家公司）= 量不够。甜区是 LinkedIn 能搜出 30–500 家。

④ 输出 / 交付物

  • 1–2 个 ICP 画像（含 5 维度）
  • 决策人地图 (3 类角色 + 关心点)
  • 3 个痛点假设
  • 验证证据（LinkedIn 搜出 ≥30 家 + 海关有进口记录）

⑤ 完成标准

⑥ 常见卡点

  • ICP 太宽（"任何对此感兴趣的"）= 群发模板诞生地。
  • ICP 太窄（只 5 家公司）= 触达不够规模。
  • 没考虑决策人 = 找到公司却找错人。
  • 没验证存在 = 设计 ICP 后发现市场没这类客户。
  • 把"理想"当"现实" = 客户根本不存在或买不起。

P02 多渠道找线索

02 多渠道找线索

把 ICP 转成名单。单渠道天花板低，组合 3–5 个渠道 ROI 最高。预计 1 周内出 ≥50 家候选池。

① 输入 / 前置

  • 01 已定义清楚的 ICP。
  • 已确认 ICP 在目标市场存在。
  • CRM 字段已建好（来源 / 公司 / 联系人 / 信号 / 评分）。

② 关键动作

  1. 按品类 + 市场选 3–5 个最匹配的渠道（见 A 卡矩阵）。
  2. 每渠道日均产出 8–12 家候选公司。
  3. 每家标渠道来源 + 搜索 query 原文。
  4. 立刻进 CRM，不让候选堆在 Excel / 笔记里。
  5. 本阶段不做联系人补全（留给 04）。

③ 渠道矩阵参考（点击进入对应 SOP）

消费品 / 视觉品类 → A01 Google + A03 IG/TikTok + A11 Maps + A06 RFQ B2B 设备 / 机械 → A01 Google + A02 LinkedIn + A07 海关 + A08 展会 工程 / 项目类 → A08 展会 + A13 招投标 + A12 合作伙伴 + A09 竞品反查 区域批发 / 小 B → A04 FB 群 + A11 Maps + A05 YouTube + A01 Google 大客户 / 决策周期长 → A07 海关 + A02 LinkedIn + A09 竞品 + A14 新闻事件 验证回报率：每周记录每渠道的候选数 + 后续回复率，淘汰底部

④ 输出 / 交付物

  • 候选公司池 ≥50 家
  • 每家有渠道来源标签
  • 每家有 query / URL 原文存档
  • 渠道分布报告（哪个渠道贡献多少）

⑤ 完成标准

⑥ 常见卡点

  • 只用单一渠道 → 池子偏 + 渠道见顶就断粮。
  • 不分优先级 → 全员都 50/50 处理。
  • 不记 query → 复盘没法重建路径。
  • 求"多"不求"准" → 候选池 200 家但只 10 家匹配。
  • 本阶段就开始触达 → 跳过验证 / 联系人补全 = 信号弱。

P03 验证需求信号

03 验证需求信号

把"可能采购"变"现在采购"。没信号的开发信是垃圾邮件。预计单家 5–15 分钟。

① 输入 / 前置

  • 02 出的候选池。
  • 清晰的"信号清单"（什么算强 / 中 / 弱）。
  • 信号工具可用（海关 / Google Alerts / LinkedIn）。

② 关键动作

  1. 每家公司过一遍信号清单。
  2. 找到信号 → 记证据链接 + 时间戳。
  3. 标强 / 中 / 弱 / 待验证。
  4. 弱 / 无信号客户先搁置（不直接淘汰）。
  5. 强信号客户 → 加速进入 04 补联系人。

③ 信号清单 + 工具（点击复制）

强信号：海关 6 个月内进口 / 招聘采购 / 主动询盘 / 新店开张 / 融资落地 中信号：官网近期更新 / LinkedIn 公司动态 / 新品发布 / 同类内容互动 弱信号：有官网 + 在做相关业务（默认状态，不算信号） 工具：A07 海关 / A02 LinkedIn 招聘 / A14 Google Alerts / Wappalyzer 网站更新 证据格式：信号类型 + 时间 + 链接 + 1 句话摘录

④ 输出 / 交付物

  • 候选池按强 / 中 / 弱 / 待验证分层
  • 每家强信号有 ≥1 条证据链接
  • 强信号客户优先级排序

⑤ 完成标准

⑥ 常见卡点

  • 把"有官网"当信号 → 信号失真。
  • 不查信号时效 → 用 3 年前的招聘当"现在采购"。
  • 不写来源 → 之后无法引用做触达上下文。
  • 强弱标准飘忽 → 评分不可比。
  • 跳过这一步直接触达 → 命中率断崖。

P04 补全联系人

04 补全联系人

没 named contact = info@ 群发 = 进垃圾箱。这一步决定回复率上限。预计单家 5–10 分钟。

① 输入 / 前置

  • 03 标了强 / 中信号的客户。
  • 已知决策人画像（来自 01 决策人地图）。
  • 邮箱抓取 + 验证工具可用。

② 关键动作

  1. A02 LinkedIn 找决策人 → 拿姓名 + 职位。
  2. Hunter / Snov / Apollo 抓邮箱（按公司域名 + 姓名生成）。
  3. NeverBounce / Hunter Verifier 验证 → 必须 valid。
  4. 补 WhatsApp / LinkedIn URL（多渠道备份）。
  5. 记下联系人在职时长（< 3 月可能没采购权）。

③ 工具栈 + 邮箱模式（点击复制）

找人：A02 LinkedIn Sales Navigator + Apollo + Lusha + RocketReach 抓邮箱：Hunter.io / Snov.io / Apollo / FindThatLead 验证邮箱：NeverBounce / Hunter Verifier / ZeroBounce 常见邮箱格式：first.last@ / firstlast@ / first@ / f.last@（用 Hunter 看公司主流模式） WhatsApp 验证：wa.me/号码 → 看是否能跳转 不要 info@ / sales@ / contact@ 作为 named contact

④ 输出 / 交付物

  • 每强信号客户 ≥1 named contact
  • 姓名 + 职位 + 在职时长
  • 邮箱（已验证 valid）+ WhatsApp + LinkedIn URL
  • 邮箱不通的备选触达路径已规划

⑤ 完成标准

⑥ 常见卡点

  • info@ / sales@ 当作 named contact → 回复率清零。
  • 不验证邮箱 → hard bounce 损害域名信誉。
  • 只一个渠道 → 邮箱挂掉就完全失联。
  • 不查在职时长 → 联系人刚走 / 刚来都没用。
  • 这一步偷懒 → 后面 06/07 全失效。

P05 CRM 入库

05 CRM 评分入库

这一步决定后续能不能复盘。字段不统一 = 数据废纸。预计单家 3–5 分钟。

① 输入 / 前置

  • 02–04 产出的客户 + 信号 + 联系人。
  • CRM 字段标准已建（来源 / 匹配 / 联系人 / 信号 / 评分 / 优先级）。
  • 评分公式已确定（多维加权）。

② 关键动作

  1. 按统一字段建档（不让信息散在邮箱 / 聊天）。
  2. 算评分：ICP 匹配 + 信号强度 + 联系人完整 + 订单潜力。
  3. 标优先级 A / B / C（基于评分阈值）。
  4. 检查重复客户合并（公司名 / 域名 / 邮箱去重）。
  5. 设跟进提醒（A 级 24h 内、B 级 72h 内）。

③ 评分公式 + CRM 字段（点击复制）

评分 = ICP 匹配 (0-30) + 信号强度 (0-30) + 联系人完整 (0-20) + 订单潜力 (0-20) A 级 ≥70 / B 级 50-69 / C 级 30-49 / 30 以下不开发 必填字段：公司 / 国家 / 行业 / 客户类型 / 来源渠道 / 信号 / 联系人 / 评分 / 优先级 / 下一步动作 工具：HubSpot Free / Pipedrive / Notion / Airtable / 自建表格 去重：公司域名 + 主要邮箱 双键判重 跟进提醒：A 级 24h / B 级 72h / C 级 7 天 / Newsletter 月度

字段标准一旦定下来不要随意改。改字段 = 之前数据全部要回填。

④ 输出 / 交付物

  • CRM 全部客户建档
  • 评分 + 优先级 (A/B/C) 已标
  • 重复客户已合并
  • 跟进提醒已排

⑤ 完成标准

⑥ 常见卡点

  • 字段不统一 → 不同人录的数据没法对比。
  • 重复录入 → 同一客户被 3 个人开发，撞车。
  • 评分凭感觉 → 评分变成噪音。
  • 不维护 → 3 个月后数据全过期。
  • 信息散在邮箱 / 聊天 / 表格 → 复盘时找不回。

P06 首轮触达

06 首轮触达

第一次曝光决定后续 80%。一个模板群发等于自杀。预计单家 5–15 分钟（含个性化）。

① 输入 / 前置

  • 05 完成 CRM 入库 + 评分。
  • 触达模板库已就绪（T01–T12）。
  • 每客户的"匹配理由 / 信号引用"已写好。

② 关键动作

  1. 按客户温度选主渠道（参考 T 卡矩阵）。
  2. 个性化前 3 行（含信号引用 + 匹配理由）。
  3. 一个清晰 CTA（不是"期待回复"）。
  4. 发送 + 追踪 (open / click)。
  5. 立刻设 D+1 / D+3 / D+7 跟进提醒（接 P07）。

③ 渠道选择矩阵（点击复制）

有公司邮箱 + 决策人 → T01 邮件主 + T02 WhatsApp 轻提醒 展会 / 询盘高上下文 → T09 / T02 24h 内推进规格 视觉品类 + 社媒活跃 → T04 IG DM 引用具体内容 找到 LinkedIn 没邮箱 → T03 LinkedIn 互动 → 连接 → DM 大企业找不到人 → T08 网站表单 + T03 LinkedIn 找人 群组 / Marketplace → T05 Messenger 引用上下文

④ 输出 / 交付物

  • 每高优客户首触发出
  • CRM 记发送时间 + 模板版本 + 渠道
  • 跟进提醒已排
  • 追踪开启（open / click 监控）

⑤ 完成标准

⑥ 常见卡点

  • 一个模板群发 → 进垃圾 + 触发反垃圾。
  • 不分客户温度 → 冷热客户用同样话术。
  • 不记发送时间 → 跟进时点不对。
  • 没跟进计划 → 首触发出就忘了。
  • CTA 模糊（"期待回复"）→ 无法推进。

P07 三轮跟进

07 三轮跟进

80% 客户不在第一次回复。三轮跟进决定胜负，但每轮要换角度。预计单客户 3 轮总共 10–15 分钟。

① 输入 / 前置

  • 06 首触已发出。
  • 跟进时间表已排（D+1 / D+3 / D+7）。
  • 第二、三轮的角度 / 价值点已准备。

② 关键动作

  1. D+1：WhatsApp / LinkedIn 轻提醒（不重复邮件内容）。
  2. D+3：补一个具体价值点（案例 / 数据 / 行业洞察）。
  3. D+7：礼貌收尾 + 问"是否正确负责人"。
  4. 每轮换角度，不重复同一信息。
  5. 3 轮无回 → 进 T11 Newsletter 长期培育。

③ 三轮节奏模板（点击复制）

A06 平台询盘

B2B 平台 / RFQ

客户主动发出的高意向信号，时效极短，半小时回复率决定胜负。预计单条 RFQ 处理 15–30 分钟（含建档 + 首次报价摘要）。

① 什么时候用

  • Alibaba / Made-in-China / Global Sources / 行业垂直平台买家请求。
  • 客户主动从产品页发起询盘 / 询价。
  • 新市场快速测试报价竞争力。
  • 展会平台 / 海关数据平台附带的买家询盘模块。

② 操作步骤

  1. 开后台筛 RFQ，按上次刷新时间倒序。
  2. 用筛选标准（见 ④）30 秒判断是否值得花时间。
  3. 30 分钟内回首条，目标是建立对话不是报价。
  4. 同步建 CRM 档，附 RFQ 截图（平台数据会被覆盖）。
  5. 拿到外部联系方式后立刻转 T01 / T02 跟进。
⚠ 避免
  • 平台站内信限流 / 关键词屏蔽，留外部联系方式可能违规扣分。
  • RFQ 时效极短，超过 2 小时回复率断崖。
  • 同一买家同时发给 5-10 家，价格战陷阱要小心。
  • 买家 1 星 / 新注册 / 主图模板化 → 假询盘比例高。
  • 不要在第一封就报全价，先确认规格再走议价。
  • 有些平台把 RFQ 数据当付费功能，做高质量回复但别盲目下注。

③ RFQ 筛选标准 + 首回模板（点击复制）

数量明确 + 规格清晰 + 采购周期 ≤90 天 + 国家产品对得上 买家等级 ≥3 星 / 询盘历史 ≥3 条 / 主图非默认头像 首回开头：Thank you for the RFQ on [产品]. Quick clarification before we quote — what [关键参数] do you need? 首回结尾：Happy to share full quote + samples once we confirm specs. WhatsApp / email also available. 把高频问题做成"资料包"：MOQ + 认证 + 交期 + 起订价格段 + 案例 3 张图

买家通常同时收 5–10 个报价，第一条比最低价更重要。先用一个专业问题"截胡"对方注意力。

A07 采购事实

海关与进口数据

最硬的需求验证渠道：客户是否真采购、买谁的、买多少、买多久。开发信里的事实依据。预计单家分析 10–20 分钟。

① 什么时候用

  • 验证客户是否真采购过同类产品（不是看官网猜）。
  • 反查竞品在某市场的客户和价位段。
  • 找进口商 + 估算市场规模 + 锁定大客户。
  • 判断客户的供应商是否在换、是否扩量、是否进了新品类。

② 操作步骤

  1. 查 HS 编码（用客户产品反推 → CN/US/EU 海关查询）。
  2. 选国家 + 时间窗口（建议最近 24 个月）。
  3. 导出进口商列表，按金额 / 频次排序。
  4. 判断供应商变化（最近 6 个月有否换 / 新增）。
  5. 用 Google + LinkedIn 补 contact，建档进 CRM。
⚠ 避免
  • 中国、欧盟、日本部分国家无公开海关数据。
  • 公司名可能是货代 / 报关行，不是真客户 —— 看目的港和地址。
  • 数据有 3-6 月延迟，不要把"最近"理解为"现在"。
  • 触达里直接说"我看到你进了多少多少"会触发对方合规警觉。
  • HS 编码同一产品在不同国家分类差异大，要交叉验证。
  • 第三方平台费用高（年订阅 ¥6k–¥30k），先按市场选投入。

③ 工具 + 分析字段（点击复制）

主流工具：Importgenius / Panjiva / Tendata / 易之家 / Volza 免费起步：US Customs (importgenius free trial) / Indian Customs / Mexico SIAT 关键字段：HS 编码 + 进口商 + 供应商 + 单笔金额 + 总频次 + 目的港 趋势字段：最近 6 个月 vs 前 6 个月 → 增长率 / 供应商稳定度 触达切入（合规版）：We saw [品类] activity in [市场] growing — would 2-3 alternative quotes help benchmark? 不要写："我看到你 2025-03 进口了 50000 USD..."（合规红线）

数据通常有 3-6 个月延迟。中国、欧盟严格管控公开数据，需要第三方付费平台。

A08 行业活跃

① 什么时候用

  5. 建会后跟进时间表（D+1 / D+3 / D+7）。
⚠ 避免
  • 名录里的联系方式常是前台 / 公关，不是采购负责人。
  • "全球买家名录"水分大，到场未必是真采购方。

A09 竞品反查

竞品客户 / 供应链反查

最高效的 ICP 来源——竞品已经替你筛过一遍。但触达要克制：客户对"挖墙脚"敏感，必须包装成替代 / 补充而非对抗。预计单家竞品反查 1–3 小时。

① 什么时候用

  • 进入新市场摸竞品分销网络。
  • 客户在沟通中提到竞品名 → 反查同类。
  • 寻找区域代理 / 经销商网络。
  • 判断客户是否在多供应商策略（可补充而非替换）。

② 操作步骤

  1. 列 3–5 家直接竞品（同价位 / 同市场）。
  2. 翻竞品官网 → Dealer Locator / Where to Buy / Case Studies。
  3. 用海关数据反查竞品进出口和分销网络。
  4. Google "[竞品名] distributor" / "[竞品名] reviews"。
  5. 判断关系类型（独家 / 多品牌 / 项目级），决定是否触达。
⚠ 避免
  • 独家代理通常有排他协议，直接挖墙脚可能引发法律风险。
  • 触达不要攻击竞品质量 / 价格，反而显得不专业。
  • 大客户多供应商是常态，定位"第二来源"接受度高。
  • 抄走竞品官网客户清单群发 → 100% 被察觉 + 行业口碑差。
  • 不要伪装成"前竞品同事"套话——查证一次就翻车。
  • 客户提竞品好话 → 加分；提坏话 → 你别接话只听。

T08 低打扰

网站表单 / 在线客服

① 什么时候用

  • Google / LinkedIn / 海关找不到具体负责人。
  • 大企业官网有 "Become a Supplier" / "Wholesale" / "Partner" 入口。
  • 邮件 hard bounce 后的备选路径。
  • 招投标 / 政府采购的预沟通通道。

② 操作步骤

  1. 找正确入口（Contact / Wholesale / Suppliers / Partners）。

  • 公司类型 / 国家选错 → 自动分流到错误部门。
  • 自动回复邮箱常无效（noreply@）→ 准备备选触达。

  • 错误电话 / 错别字 → 直接被视为不专业。

T09 现场上下文

外贸最高浓度的上下文来源。3 天展会的名片 = 6 个月 cold outreach 的总量。但黄金期只有 14 天，过了 = 80% 失效。预计单张名片 5–10 分钟跟进。

① 什么时候用

  • 展会现场即时交流后。
  • 会后 24h-14 天的黄金跟进窗口。
  • 客户留下名片 / 留资 / 样品咨询。
  • 现场没谈完的预约 / 工厂参观邀请。

② 操作步骤

  1. 现场记 3 个关键词（产品 + 需求 + 承诺动作）。
  2. 名片当场拍照 + OCR 入库 + 标记优先级 (A/B/C)。
  3. 当天 / 次日发首条消息引用具体细节。
  4. 24h 内发资料摘要 + 下一步选项。
  5. D+3 / D+7 / D+14 三轮跟进序列。
⚠ 避免
  • 名片堆积 ≥7 天 → 回复率断崖（黄金期错过）。
  • 现场不记笔记 → 一周后认不出谁是谁。
  • 同一模板群发"很高兴在展会见到您" → 客户秒识破。
  • 把所有人都标 A 级 → 优先级失效 + 资源浪费。
  • 承诺"回去发资料"超 24h 未兑现 → 信任清零。

T10 区域习惯

区域市场的"本地通讯"补充渠道。WhatsApp 不通 / 客户已习惯本地工具时启用。合规边界各国差异极大，不能套用单一逻辑。预计单条 3–5 分钟。

① 什么时候用

  • 客户已给号码 / 已有轻度上下文（不是冷开发）。
  • 区域市场本地工具优势（俄 / 中东 → Telegram；日台泰 → Line；菲 → Viber；华人圈 → WeChat）。
  • WhatsApp 在当地受限或客户主动建议换工具。

② 操作步骤

  1. 确认对方当地常用工具 + 合规边界。

⚠ 避免

T11 长期培育

① 什么时候用

  • 客户未到采购周期但匹配度高。
  • 需要长期品牌信任（高客单 / 项目级 / 长周期）。
  • 沉睡线索（6 个月以上无互动）激活。
  • 季度新品 / 认证更新 / 市场动态发布。

② 操作步骤

  1. 设计内容历（案例 / 认证 / 趋势 / FAQ / 选型）。

⚠ 避免

  • 不分群一刀切 → 内容不相关 → 客户标垃圾。

  • 域名信誉低 / 大量 hard bounce → IP 被运营商屏蔽。
  • GDPR / CASL / CAN-SPAM 必须 opt-in + 实际地址。
  • 把废邮箱 / 退订名单继续推 → 罚款 + 域名永久受损。

T12 信任背书

转介绍触达

回复率最高的触达类型（带信任背书直接跳过 80% 警觉）。但要保护推荐人信誉——一次失败影响下次转介绍。预计单条 5–10 分钟（含闭环）。

① 什么时候用

  • 老客户主动 / 索取式推荐。
  • 代理 / 物流 / 检测 / 行业朋友的同业引荐。
  • 展会 / 群里二度连接的间接介绍。
  • LinkedIn 共同好友引荐。

② 操作步骤

  1. 第一条说明推荐人 + 推荐理由（前 3 行内）。
  2. 简介身份 + 资料匹配对方业务（不发泛用目录）。
  3. 一个问题（不直接卖货）。
  4. D+1 / D+3 / D+7 跟进可以更快但仍克制。
  5. 闭环感谢推荐人（成与不成都要反馈）。
⚠ 避免
  • 推荐人名字拼错 = 信任清零，推荐人下次不帮。
  • 不闭环 = 推荐人不知道结果，下次不愿介绍。
  • 用泛用目录代替针对性资料 = 浪费推荐信任值。
  • 不要在推荐人面前批评被推荐人。
  • 推荐礼承诺要兑现，否则信用透支。
  • 跟进太密集 = 即使是推荐人介绍也会反感。

评分公式一旦定下来不要随意改。改公式 = 之前所有客户评分要重算。

⑤ 常见错误

  • 风险点空 → 后期被坑没预案。
  • ICP 匹配理由是"做相关产品" → 触达时无具体可引用。
  • 所有客户都标 A 级 → 优先级失效。
  • 信号未验证就标"强" → 转化率打脸。
